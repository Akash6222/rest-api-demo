# REST API Demo

A production-ready REST API demo built with Python 3.13 and FastAPI.

## Features

- **FastAPI**: High performance web framework.
- **Pydantic**: Data validation and settings management.
- **Docker & Docker Compose**: Ready for deployment (e.g. Dokploy).
- **Environment Variables**: Configured using `.env`.
- **CORS Support**: Configurable Origins.
- **Logging**: Configured standard Python logging.
- **Graceful Error Handling**: Global exception handling.
- **Health Checks**: Standard health check endpoints.

## Project Structure

```
rest-api-demo/

│   ├── config.py        # Environment variables and configuration

└── README.md
```

## Installation & Local Development

1. **Clone the repository** (or navigate to the directory):
   ```bash
   cd rest-api-demo
   ```

2. **Create a virtual environment** and activate it:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

3. **Install dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Set up environment variables**:
   ```bash
   cp .env.example .env
   ```

5. **Run the application**:
   ```bash
   uvicorn app.main:app --reload
   ```
   The API will be accessible at [http://localhost:8000](http://localhost:8000).

## Docker Commands

This project is built to be deployed seamlessly with Docker and Dokploy.

### Running with Docker Compose

1. **Build and start the container in detached mode**:
   ```bash
   docker compose up -d
   ```

2. **View logs**:
   ```bash
   docker compose logs -f
   ```

3. **Stop the container**:
   ```bash
   docker compose down
   ```

### Running with plain Docker

1. **Build the image**:
   ```bash
   docker build -t rest-api-demo .
   ```

2. **Run the container**:
   ```bash
   docker run -d -p 8000:8000 --name rest-api --env-file .env rest-api-demo
   ```

## API Endpoints

Once running, interactive API documentation is automatically generated by FastAPI:
- **Swagger UI**: [http://localhost:8000/docs](http://localhost:8000/docs)
- **ReDoc**: [http://localhost:8000/redoc](http://localhost:8000/redoc)

### System Endpoints
- `GET /` - API Information
- `GET /health` - Health check status
- `GET /version` - API version

### User Endpoints
- `GET /users` - List all users
- `GET /users/{id}` - Get a user by ID
- `POST /users` - Create a new user
- `PUT /users/{id}` - Update an existing user
- `DELETE /users/{id}` - Delete a user

## Example curl Commands

**Get API info:**
```bash
curl http://localhost:8000/
```

**Check health:**
```bash
curl http://localhost:8000/health
```

**Get all users:**
```bash
curl http://localhost:8000/users
```

**Create a new user:**
```bash
curl -X POST "http://localhost:8000/users" \
     -H "Content-Type: application/json" \
     -d '{"name": "Jane Doe", "email": "jane@example.com", "is_active": true}'
```

**Update a user:**
```bash
curl -X PUT "http://localhost:8000/users/1" \
     -H "Content-Type: application/json" \
     -d '{"name": "Alice Wonderland"}'
```

**Delete a user:**
```bash
curl -X DELETE "http://localhost:8000/users/2"
```
